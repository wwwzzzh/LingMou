# 灵眸AI 设计文档

> AI 视觉对话助手 — 基于多模态大模型的实时视觉语音助手

---

## 一、项目背景

灵眸AI 是一个端云协同的实时视觉语音助手。用户通过摄像头和麦克风与 AI 进行多模态对话。前端 Vue3 + TypeScript，后端 Spring Boot 3.x，AI 能力接入火山引擎豆包模型。

## 三、成本控制策略

### 3.1 考虑过的方案

| 编号 | 策略 | 描述 | 预期节省 |
|------|------|------|:--:|
| CS-01 | 画面变化检测 | 静止画面不上传图片 | 70-90% |
| CS-02 | 图片压缩缩放 | 降低图片分辨率（≤640px）和质量（Q0.7） | 50-80% |
| CS-03 | VAD 静音检测 | 用户不说话时不发送音频 | 30-50% |
| CS-04 | 低频检测 | 主动监控 10s 间隔定时拍照，非全帧率 | 99%+ |
| CS-05 | 端侧 ASR | 浏览器 Web Speech API，免费 | 100% |
| CS-06 | 端侧 TTS | 浏览器 SpeechSynthesis，免费 | 100% |
| CS-07 | 对话轮次限制 | 保留最近 20 轮，超出的压缩或丢弃 | Token 节省 |
| CS-08 | 请求限流 | Redis 滑动窗口，60s 最多 20 次 | 防滥用 |
| CS-09 | 主动监控冷却 | 变化报告后 20s 冷却，避免重复 API 调用 | 90%+ |
| CS-10 | 模型分级 | 主对话用 vision-pro，纠错用 seed-lite | 约 30% |
| CS-11 | 本地部署 | Ollama 本地模型，完全离线 | 100% |
| CS-12 | 端侧描述 | 不调 AI，纯客户端描述变化 | 100% |

### 3.2 实际采用

| 编号 | 策略 | 状态 | 实现位置 |
|------|------|:--:|------|
| CS-01 | 画面变化检测 | ✅ | motionDetector.ts：120px 缩略图 RGB 采样 |
| CS-02 | 图片压缩缩放 | ✅ | ChatView handleSend：Canvas 缩放 + JPEG 0.7 |
| CS-03 | VAD 静音检测 | ⚠️ | vad.ts 已实现，ChatView 未集成 |
| CS-04 | 低频检测 | ✅ | useActiveVision.ts：10s 间隔定时拍照 |
| CS-05 | 端侧 ASR | ✅ | useSpeechRecognition.ts：Web Speech API |
| CS-06 | 端侧 TTS | ✅ | useSpeechSynthesis.ts：SpeechSynthesis |
| CS-07 | 对话轮次限制 | ✅ | 后端 ChatSessionService |
| CS-08 | 请求限流 | ✅ | RateLimitInterceptor |
| CS-09 | 主动监控冷却 | ✅ | useActiveVision.ts：20s COOLDOWN |
| CS-10 | 模型分级 | ✅ | vision-pro 主对话 + seed-lite 纠错 |
| CS-11 | 本地部署 | ❌ | 硬件要求高（32GB+4060），未采用 |
| CS-12 | 端侧描述 | ❌ | 准确度不足，采用 AI 分析更可靠 |

> **12 项策略中完全采用 10 项，部分采用 1 项，未采用 2 项。**
